Download and unzip sample_dir.zip. Use the cd command to navigate to your downloads folder and enter the newly unzipped Quinnipiac directory.

  1. TH130
  2. LearningCommons
  3. CCE030
  4. Quinnipiac (return to the starting directory)

  1. What's the smallest number of commands you could have used to visit all of these locations?
  2. What's the largest number of commands you could have used to visit all of these locations? For a command to count, it must take you closer to the next location you're trying to visit.

Exercise 2: Exploring Java

Below are three example Java programs. Copy each program to a new file in your workspace and try running it. If you're unable to run a program, then ask for help! All programs should work as long as you're using Java 23 with preview features enabled.

Program 1: Hello.java

void main() {
    println("Hello Zach, welcome to CSC110!");
}

Program 2: FullName.java

void main() {
    var firstName = readln("What is your first name? ");

    var lastName = readln("What is your last name? ");

    println("Your full name is " + firstName + " " + lastName + ".");
}

Program 3: Add.java

void main() {
    var x = Integer.parseInt(readln("Please enter an integer. "));

    var sum = x + 1;
    println(sum);
}

Once you've run each of the programs, try making the following changes:

  1. Make Hello.java greet you instead of your professor.
  2. Make FullName.java print the user's name in "Last, First" format instead of "First Last". For example, "Kohlberg, Zach" instead of "Zach Kohlberg".
  3. Make Add.java read two numbers from the user and print their sum instead of reading a single number and printing that number plus one.

We haven't explained what this code does yet, but I want you to try figuring it out based on the working examples. It's okay if this takes some trial and error. We'll learn more about how these programs work next week.

Week 3

Write the programs in these practice exercises from scratch, and try to avoid looking at and copying from previous examples. It's a little more work, but they're short programs and it'll help you remember how these simplified Java program are structured.

Problem 1: Hard-Coded Email

Write a Java program that prints your QU email address. It only has to print your email, not anyone else's email. Run the program on your terminal to make sure it works.

Problem 2: Email Script

Pair up with another student for this exercise. Each of you need to figure out the other person's QU email address, and you're not allowed to show or tell each other your email address. Instead, come up with a script that you can follow to ask someone for the information necessary to figure out their QU email. Test the script on each other and make sure it works.

Problem 3: Variable Email

Write a Java program that prints the user's QU email address. As with problem 2, you can't just ask them for their email and repeat what they tell you. You'll need to prompt the user for information that can be used to determine their QU email, then print that email to the terminal. Your script from problem 2 should help.

Remember that this program must print the user's email, and people other than you might use it. When you finish this program, anyone in the classroom should be able to use it to print their email without modifying your code. Have a classmate try running it to make sure it also works for them.

Hints:

  • prompt for input with readln("some message in quotes")
  • always store input in a variable
  • you can combine variables with quoted text by placing a plus sign between them, as shown in previous examples

Part 1: Expression Types

Look at the following expressions and, without typing them into jshell or a java program, try to predict both the type (int, double, char, or String) and value of each expression. For example, the first expression results in the int 4.

Once you've made your predictions, try running each expression in jshell to check your answers. Note which predictions were wrong and try explaining why the answer didn't match your prediction. If you're not sure, discuss it with a classmate.

  1. 2 + 2
  2. 2 + 2.0
  3. 2 + "2"
  4. 2 + '2'

  6. 10 % 3
  7. 2.0 / 3
  8. 0.1 + 0.1 + 0.1
  9. "A" + 2 + 2
  10. 2 + 2 + "A"
  11. "A" + "B"
  12. 'c' - 'a'

Part 2: Short Programs

Write a java program to solve each of the problems listed below. You are not allowed to use the var keyword in any of these programs; you must explicitly state your variable types.

  1. Ask the user for a temperature in Fahrenheit and convert that temperature to Celsius. (F = C * 9 / 5 + 32)
  2. Same as the previous problem, but convert from Celsius to Fahrenheit. (C = (F - 32) * 5 / 9)
  3. Ask the user for their height in inches, then display their height in feet and inches. For example, if the user said they were 70 inches tall, then your program would tell them that they are 5 feet and 10 inches tall. Don't worry about grammar, it's okay for your program to print results such as "1 feet and 0 inches" even if we wouldn't normally phrase it this way. Hint: you can get the leftover inches by dividing the total inches by 12 and taking the remainder instead of the quotient. Do you remember an operator that gives you a remainder?

Part 3: Functions

For each question listed below, write a function that returns the answer to that question. You should avoid using print, println, and readln in these functions; instead, declare parameters for the input and return the output. You can call the functions with hard-coded inputs and print their results in your main function to test them.

  1. Given a temperature in Fahrenheit, what is the equivalent temperature in Celsius?
  2. Given a temperature in Celsius, what is the equivalent temperature in Fahrenheit?
  3. Given a person's full height in inches, how many feet would you write when stating their height in feet and inches?
  4. Given a person's full height in inches, how many inches would you write when stating their height in feet and inches?

If you finish all of the problems early, then try modifying the programs from Part 2 to use the functions you wrote in Part 3.

Part 3: Problem Solving

Design a flowchart to solve each problem described below. After you create the flowchart, try following the flowchart directions with pen and paper, and write the program if you have enough time after testing the flowchart.

  1. Prompt the user for an integer. If they enter a multiple of 5, then tell them that the number is a multiple of 5. If not, then tell them the remainder of their number when divided by 5.
