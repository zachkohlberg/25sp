Notes for Learning to Program

Java

Java is the programming language we're learning. We're also using some newer preview features to make it easier to get started, which will require Java 23 or newer. To develop in Java, we need the Java Development Kit (or JDK). We'll use the implementation provided by the Eclipse Foundation, called Java Temurin.

Install

These links include filters to only show jdk 23 for the specified operating system. Download and run the appropriate installer.

  • MacOS (download the .pkg): https://adoptium.net/temurin/releases/?version=23&os=mac&package=jdk
  • Windows (download the .msi): https://adoptium.net/temurin/releases/?version=23&os=windows&package=jdk

On Linux I recommend installing java 23 with your package manager if it's available. If not, then you can try manually installing the .tar.gz archive ( https://adoptium.net/temurin/releases/?version=23&os=linux&package=jdk) or following the directions on the installation guide (https://adoptium.net/installation/).

Check Version

After installing Java 23, open your terminal (if it's already running, then close and reopen it). Run the following commands and make sure they show Java 23:

java -version
javac -version
jshell -version

For example, my output looks something like this:

$ java -version
openjdk version "23" 2024-09-17
OpenJDK Runtime Environment (build 23+37-2369)
OpenJDK 64-Bit Server VM (build 23+37-2369, mixed mode, sharing)
$ javac -version
javac 23
$ jshell -version
jshell 23

Java Alias

All of our programs will be using some preview features from Java 23 to remove some of the boilerplate code that we'd normally have to include in every Java program. I may also make use of asserts in my examples. Both asserts and preview features are disabled by default when running the java command. You must include the -ea and --enable-preview flags to enable them.

You can manually type those flags every time you run a java program (java -ea --enable-preview), or you can create an alias to always include them with the java command. I recommend creating the alias in one of the scripts that your terminal runs at startup. You can do this with one of the commands below:

For any operating system using a bash terminal (on MacOS, echo $0 displays bash):

echo "alias java='java -ea --enable-preview'" >> ~/.bash_profile

For MacOS using a zsh terminal (echo $0 displays zsh):

echo "alias java='java -ea --enable-preview'" >> ~/.zprofile

I recommend copying these commands (hover your mouse on the command and click the copy button that appears on the right side) and pasting them into your terminal rather than typing them by hand. If you mistype the command, then it may fail to work or overwrite existing settings if you already have a .bash_profile or .zprofile file.

After adding the alias, close and reopen your terminal. Run the command alias java and make sure it prints the alias shown in the above commands. If it does, then you're all set!
