Week 3

Types

The type of a variable or expression tells us what kind of information it represents and how we're allowed to use it. For now, the types we're going to learn about fall into four groups: integers, real numbers, text, and boolean.

All information in a computer is represented in binary, a number system that only uses the digits 0 and 1. A single binary digit is called a bit, and we can measure the amount of memory a type uses by the number of bits required to encode it.

Integers

Integers include positive and negative whole numbers (and zero). Java has several integers types: long, int, short, and byte. Each integer type takes up a different amount of memory and can store a different range of numbers. You can see the details in the types table further down.

The smaller integer types allow programs to use less memory when dealing with many small numbers, and the larger integer types allow programs to represent much larger numbers. By default, Java uses the int type for integer values, and this is usually enough for our purposes.

Java also has a BigInteger type to handle arbitrarily large integers, but this is used much less often than the other integer types. It takes up more memory and is slower to use in computations.

Real Numbers

Real numbers include integers, fractions, and numbers like \( \pi \) that cannot be represented as a fraction. Java has two types that can represent real numbers: float and double. The name "float" refers to floating-point, the system used to represent these numbers in binary. The name "double" is short for "double-precision floating-point number."

Floating numbers can get quite large. The minimum and maximum values in the types table are represented using a form of scientific notation called E notation. You'll see E notation used if your program tries to print a particularly large or small floating-point number. If we were to write 3.4028235E38 without scientific notation, it would be 340,282,350,000,000,000,000,000,000,000,000,000,000.

Question: You may have noticed the both an int and a float use 32 bits of memory, but the float type can represent a far greater range of values than the int type. Shouldn't this make int obsolete? Do a bit of research and try to find out why we still use types like int and long instead of always using float and double. We'll cover this in more depth when we learn binary.

Like the integer types, there is also a special BigDecimal type that can store real numbers of an arbitrary size. It has similar trade-offs to the BigInteger type.

Text

We use two different types to represent text: char and String. A char is a single character, such as a letter, digit, or punctuation mark, and a String is a sequence of characters.

Boolean

The boolean type doesn't fit in any of the other categories. It can represent just two values: true and false. We'll learn more about booleans when we discuss control flow and how our programs make decisions.

Uppercase Types

You may notice that the last three types on that list all begin with an uppercase letter, unlike the others. Types that begin with an uppercase letter have additional capabilities called methods. A method is a function attached to a piece of data, like a string. For example, a string can convert itself to all uppercase letters:

void main() {
    println("hello world".toUpperCase());
}

Arithmetic

For the most part, addition, subtraction, multiplication, and division work as you're used to. The main exception is integer division: when you divide one integer by another integer, the result will always be an integer. This means 7 / 3 results in 2, not 2.33333....

Modulo

Unless you already have programming experience, you probably aren't familiar with the modulo operator. This operator divides two numbers and gives the remainder rather than the quotient. It's related to modular arithmetic, and it has a surprising number of uses in programming.

Concatenation

If you attempt to add a string to a value of any type, your program will perform string concatenation instead of addition. Concatenation converts the other value to a string (if it wasn't already) and combines them into a single string. The expression "Hello" + " " + "World", for example, would result in the string "Hello World".

Variables

When we introduced variables, we used var to declare a new variable. This tells Java to infer the variable's type based on the initial value we provide. We can also state the variable's type instead of using var:

var x = 10; // infers x is an int
int y = 5;  // explicitly states y is an int

We used var initially so that we could introduce variables without also covering types, but I would recommend avoiding var and stating the type for most variables.

Functions

Now that we've learned about types, we can talk more about functions. For starters, look at the following example:

void main() {
    println(square(5));
}

int square(int x) {
    return x * x;
}

This example demonstrates two concepts that we left out when we first discussed functions: parameters and return values.

When you define a function, you can list parameters in the parenthesis after its name to require additional input when the function is called. Each parameter must have a type followed by a name, just like a variable declaration. This is the int x in the definition for the square function in the above program.

You can also write functions that return information to the code that called them. This is how the readln function is able to give you the string it reads from the user. If a function returns a value, then it must state the type of value it returns before its name. Functions that don't return a value instead put void before their name.
