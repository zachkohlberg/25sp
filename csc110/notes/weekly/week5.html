Concatenation Versus String.format

So far we've been formatting our output strings by concatenating string literals and variables. This can lead to some verbose expressions that are difficult to read:

void main() {
    int x = Integer.parseInt(readln("Please enter an integer. "));
    int y = Integer.parseInt(readln("Please enter a second integer. "));
    println(x + " + " + y + " = " + (x + y));
    // if user enters 2 and 3, then the program will print "2 + 3 = 5"
}

Another way to format output is with the String.format function:

void main() {
    int x = Integer.parseInt(readln("Please enter an integer. "));
    int y = Integer.parseInt(readln("Please enter a second integer. "));
    println(String.format("%d + %d = %d", x, y, x + y);
    // if user enters 2 and 3, then the program will print "2 + 3 = 5"
}

This function takes a format string ("%d + %d = %d") and a list of expressions (x, y, x + y). This separates the structure (the format string) from the values we're inserting (the list of expressions), which can make your code more readable. The %d is a formatting specifier, which tells the function where to insert a value in the format string and what type of value to expect. A formatting specifier begins with a % and ends with conversion, a letter that indicates the type of value you want to display.

You can also include symbols between the % and the conversion to modify how it formats the value. For example, you can pad a number with leading zeroes or spaces, add a plus sign in front of nonnegative numbers, or limit the number of digits to include after a decimal place.

Finally, note that the percent sign has a special meaning inside of a format string. If you want to display a percent sign in the result, then you'll need to escape it with an additional percent sign.

The Formatter page of the Java API goes into detail about format strings and formatting specifiers. I'll include a few formatting specifiers below that you may find useful this semester along with some examples of how to use them. Jshell is a great tools for testing a format string to make sure it works the way you want it to. Try running the examples in jshell and modifying them to get a better feel for how they work.

Formatting specifiers:

  • %d: display a base-10 integer
  • %f: display a base-10 real number
  • %s: works with any value
  • %5d: a decimal number padded with leading spaces to occupy at least 5 characters; you can use any positive integer instead of 5, and this also works with other conversion types
  • %05d: same as the previous specifier, but pads with leading zeroes instead of spaces
  • %.2f: a real number rounded to exactly 2 digits after the decimal place; you can use any nonnegative integer instead of 2, and you can combine this with the previous two specifiers (for example, %07.3f)
  • %+d: adds a plus sign to zero and positive numbers (negative numbers still start with a minus sign); this can be used with %f and combined with any of the previous three specifiers (for example, %+5d)

Examples:

void main() {
    int x = -1, y = 0, z = 1;
    println(String.format("%+5d\n%+5d\n%+5d", x, y, z));
    double percentage = 0.7381867;
    println(String.format("%.2f%%", percentage * 100));
    int cents = 12345;
    println(String.format("$%d.%d", cents / 100, cents % 100));
    int first = 50000, second = 25000, third = 5000;
    println(String.format(
        "High Scores:\n\t1. %09d\n\t2. %09d\n\t3. %09d",
        first,
        second,
        third));
}
