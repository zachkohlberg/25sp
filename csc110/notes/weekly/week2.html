Notes for Learning to Program

Week 2

Java Programs

We're using preview features in Java 23 to simplify the structure of Java programs. With these preview features, a Java program looks like this:

void main() {
    // our code
}

The void main() creates a function called "main". When a Java program runs, it always starts by running the code in this main function. All of your programs must include this void main(). The curly brackets { ... } enclose all of the code inside of the main function: this is the code your program will run.

The program also must be saved in a file, which should be named according to the following rules:

  1. The filename must end in .java (all lowercase)
  2. The filename must not contain any spaces
  3. The filename should be written in UpperCamelCase, which means the first letter of each word is capitalized

You can run your program from the terminal by navigating to the directory where you saved the program and running the command java MyProgram.java. Replace "MyProgram" with the name of your program.

Output

Traditionally, the first program we write when learning to code is called Hello World. This is a program that prints some version of the phrase "Hello World" to the command line. A Hello World program looks like this:

void main() {
    println("Hello World");
}

We use the term "output" to describe information produced by our program. A program's output can include text on the terminal, new files or changes to existing files, images or animated graphics, sounds, and more. Most of our programs in this class will produce all of their output by printing text to the terminal, but we may have time to learn other forms of output later in the semester.

For now, we have two ways to produce output:

  • println prints a line of text to the terminal
  • print prints text to the terminal, but it does not move to a new line after printing the text

Functions

Much of the code we write will involve functions. A function is a block of code with a name attached to it, and we can run the code inside that function by calling (or invoking) the function. The Hello World program demonstrates defining a function and invoking a function:

This part defines a function called main:

void main() {
    ...
}

This part invokes a function called println:

println("Hello World");

When we define a function, we tell the computer what is supposed to happen when this function is invoked. When we invoke a function, we actually run the code inside the function. We don't have to invoke our main function because Java does this for us when it starts running the program.

Function Arguments

Some functions require additional information before they can run, just like a command you write on the terminal. When we provide this information, it is called an argument. The arguments for a function call always appear in parenthesis after the function's name.

For example, the println function needs an argument to tell it what text to display. In the Hello World program, the quoted text "Hello World" is an argument used when invoking println.

Variables

Our programs can produce information through calculations or manipulating text. If we want to use this new information in our program, then we need the program to remember and assign it a name. The name lets us refer to it in other parts of the code.

A variable is a name associated with a piece of data. That data can be a number, text, or something more complicated. We can create a variable by typing var followed by the variable's name, an equal sign, and the information we wish to store in the variable. For example:

var x = 2 + 2;

The above code creates a variable named x and assigns the result of 2 + 2 to this variable. If we later used the variable x, it would be treated as a number with the value 4. For example, the following program prints the number 4 because that's the value stored in the variable x:

void main() {
    var x = 2 + 2;

    println(x);
}

We can also reassign a variable, which gives it a different value, by using the name followed by an equal sign. Note that we do not include the var keyword when the variable already exists.

void main() {
    var x = 2 + 2;

    println(x);

    x = 7;

    println(x);
}

Input

So far we can write programs that perform calculations and produce output, but these programs aren't interactive. If we want our program to respond to the user, then they need to accept input. Input refers to any information we provide to our program, and it can include text, images, files, sounds, mouse movements, and more. For now, we'll focus on providing input by typing text into the terminal.

We have another function available that prompts the user for input. This function is called readln:

void main() {
    var name = readln("What is your name? ");
    println("Hello " + name + ", nice to meet you!");
}

If you run the program above and enter your name when prompted, then it will print a greeting that includes your name. Note that this does not involve editing the above code! Save the program exactly as shown above to a file and run it on the terminal.

The readln requires one argument: text with which to prompt the user for input. It will print this text, just as if you had called the print function, and then read whatever the user types. When the user presses enter, the program will resume, and the function will return any text typed by the user.

When a function returns information, this information will be lost unless we do something with it. If we want to remember that information and use it later in our program, then we can store it in a variable. That's what we did in the above program: we created a variable called name and set it equal to the text returned by the readln function.

Some functions don't return anything. These functions are marked as void, like our main function. The print and println functions are also void: they send information outside of the program, so there's nothing for them to return once they finish.

Comments

The last concept we'll introduce is called a comment. A comment is text in your program that is ignored when the program runs. We can use comments to document our code or leave notes for ourselves. There are two ways to write a comment in Java:

// This is a one-line comment. Any text after two forward slashes is a comment.

/*
This is a multi-line comment.
Any text between the slash-asterisk and asterisk-slash is a comment.
*/

You'll see comments throughout our example programs to help explain what the code is doing.
