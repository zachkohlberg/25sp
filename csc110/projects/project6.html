Tasks

Task 1: Determine Your Rules

Choose one of the options below.

Option 1: (Default) Yahtzee

If you choose this option, then you must complete the program by implementing the remaining parts of Yahtzee. Your program must follow all of the standard rules except for the "Yahtzee Bonus" rules in the official rules. In your program, a Yahtzee should be treated like any other scoring category with no special cases.

Program summary:

  • A single user will play a full thirteen rounds of Yahtzee.
  • Each round will consist of a reroll phase and a score phase.
    • During the reroll phase the user can reroll their dice up to two times, and they can choose which dice to reroll each time.
    • During the score phase the user must choose one unused score category to determine how many points they receive for their dice.
  • After thirteen rounds, the game will end and the user will see a final scorecard showing their points in each category, as well as their total score.

This option also requires your program to pass a set of tests. The additional details section has a list of requirements that will be checked by the tests, and the structure provided by these requirements may be helpful if you're having trouble writing this program without the usual guided coding steps.

Updating main

To help you get started, here's an outline of the main function for the default version of the program:

  • Create the following arrays, which will be passed to and used by other functions in your program:
    • an array of thirteen strings for the category names
    • an array of thirteen ints for the category scores
    • an array of thirteen booleans for the availability of these categories and set each element to true
    • an array of five ints for the dice
  • Create a loop to play thirteen rounds of the game, which should:
    • Print the round number
    • Roll the dice
    • Call rerollPhase
    • Call scorePhase
  • Print the final score card after the loop terminates.

Additional Details

This section covers the requirements for anyone who chose the default option. If you went with the custom option, then you can ignore this part of the assignment.

I/O Format

This program must meet the following I/O requirements:

  • The program accepts the required input in the order shown in the assignment and does not expect any additional input.
  • The program produces each line of required output in the order specified in the assignment.

The program may include additional output before, between, and after lines of required output. This additional output must be on a separate line from any required output.

Your program must accept the same user input as in Project 4 for every round of the game. There will be 13 rounds in a full game.

Your program must print the same output as in Project 4, but with the following changes:

  • At the beginning of each round print "Round X", where X is the round number (starting at 1)
  • During the score phase:
    • A list of options produced by this project's version of the printScore function
    • The message "ERROR: INVALID CATEGORY" when user chooses a category that isn't a number from 1 to 13
    • The message "ERROR: CATEGORY UNAVAILABLE" when the user chooses a category that was already taken
    • The number of points received for the chosen category and that category's name formatted as "You receive X points for Y."
  • After the final round:
    • The line "Final Score Card:"
    • The output of the printScore function

Additionally, your program should include reasonable prompts for information and any additional text that you think would make it easier for the user to interact with the program. It's up to you how much of this to include, what exactly to say, and how to format it. The tests will not check these prompts.

Functions

Your program must include all of the functions described below in addition to the functions required in Project 3 and Project 4. If a function from Project 3 or 4 appears below, then the version in this project replaces the version from the earlier project.

scorePhase

  • Return type: void
  • Parameters:
    • An int array representing a set of five dice
    • An int array containing the score for each category
    • A boolean array representing whether each category is available
    • A String array containing the name of each category

Must perform the following process:

  • Display the score options, their availability, and how many points each option is worth (call printScores)
  • Read integers from the user until they enter a number from one to thirteen that corresponds to an available category, displaying an error message for each invalid input
  • Calculate and print the score for the category chosen by the user
  • Store the score in the scores array
  • Mark the category as unavailable

calculateDiceScore

  • Return type: int
  • Parameters:
    • An int array representing a set of five dice
    • An int representing a score category (assume a number from 1 to 13)

The numbers 1 to 13 correspond to the following score categories:

  • 1 -> Aces
  • 2 -> Twos
  • 3 -> Threes
  • 4 -> Fours
  • 5 -> Fives
  • 6 -> Sixes
  • 7 -> Three of a Kind
  • 8 -> Four of a Kind
  • 9 -> Full House
  • 10 -> Small Straight
  • 11 -> Large Straight
  • 12 -> Yahtzee
  • 13 -> Chance

The score is calculated using the given set of dice.

You may wish to break this function into separate functions for each score category, but it's not required.

printScores

  • Return type: void
  • Parameters:
    • An int array representing a set of five dice
    • An int array containing the score for each category
    • A boolean array representing whether each category is available
    • A String array containing the name of each category

For each score category, print:

  • The category's number (1 to 13)
  • The category's name
  • If the category is available, then the word "available", followed by the score the user will receive from their dice if they choose that category
  • If the category is unavailable, then the score previously assigned to that category

The Upper Section's Bonus is 35 points if categories 1-6 add up to 63 or more points; otherwise, it is 0 points. The Upper Section Total must include the Bonus. The bonus and totals should only be based on the numbers in the scores array, not the numbers calculated for available categories.
