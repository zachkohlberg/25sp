Project 4: Dice Game Part 2

This project adds a scoring phase and reroll limit, letting us play a full round of Yahtzee. However, we will only use a subset of the normal scoring categories.

Tasks

Task 1: Guided Coding

Follow the steps below and write the program as directed. Test your code frequently, at the bare minimum performing every test given in the steps.

If your code is not working as expected, do not move on to the next step until you have fixed it. Moving on without finishing a step correctly will make the problem significantly harder to fix and likely add new problems as you continue working. If you disregard this advice, then you may get to the end of the steps and find it would actually be less work to start over from step 1 than to debug the code you just finished writing.

  1. The first version of our dice game only handled rerolling dice. This version will add a scoring phase where we choose a category and earn points from our dice. Our first step will be to move most of the code from main into a function called rerollPhase. The rerollPhase function will be responsible for letting the user reroll their dice before scoring.

    The main function should still create the dice array and call rollAllDice, but everything else can move to the rerollPhase function. This function should have one parameter for the dice array.

  2. We need to add a restriction to the reroll phase. In our finished game, the user is only allowed to reroll up to two times before scoring their dice. This means you'll need a variable to track the number of rerolls left.

    We're also going to change the "quit" command to "s" for "skip". Instead of quitting the game, this command will skip any remaining rerolls. The user is not forced to reroll their dice if they're happy with what they currently have.

  3. Write the countDice function to help you calculate the user's score. This function should iterate over an int array and count how many times a die value appears in the array.

    You can test this function in main by calling it with some sample arrays and die values:

    println(countDice(new int[] {1, 2, 3, 3, 6}, 3));
println(countDice(new int[] {4, 4, 6, 3, 1}, 5));
// and so on...

  4. We need a printScores function to display all of the score categories and how many points they're worth for a given set of dice. This function must take an int array (representing the dice) as its parameter. To start, I recommend creating the following array at the beginning of your function: String[] categoryNames = new String[] {"Aces", "Twos", "Threes", "Fours", "Fives", "Sixes"};. This will give you an array with all of the category names and make it easier to use a loop in this method.

    Write a for loop that iterates over each category index and prints the category number, name, and score. The category number is equal to the index plus one, and this is what the user will type later to select the category. The category name is the string from the categoryNames array shown above. The score is calculated by multiplying the category number by how many times it appears in the dice array (hint: use your countDice function).

    For example, if the dice included three ones, then the first line of output would be "1. Aces: 3". "Aces" is the name of the first category, its index is 0, its number is 1, and we would earn 3 points if we chose to score that category.

  5. Finally, create a function named scorePhase and call it in your main function. The function must have an int array as its only paramater, and you should pass the dice array when calling it in main.

    This function must print the score categories (use printScores), then prompt the user to choose a score category. If the user enters an invalid category number, then you must continue to prompt them until they choose a valid category number (use a while loop).

    After the user has selected a category, print the score they've earned.

Additional Details

I/O Format

This program must meet the following I/O requirements:

  • The program accepts the required input in the order shown in the assignment and does not expect any additional input.
  • The program produces each line of required output in the order specified in the assignment.

The program may include additional output before, between, and after lines of required output. This additional output must be on a separate line from any required output.

Your program must accept the following user input:

  • During each iteration of the reroll phase:
    • The command "r X", where "X" is a series of letters from a to e
    • The command "s"
  • During the score phase:
    • An integer from 1 to 6
    • If the integer is not a valid choice, then read additional integers until a valid choice is entered

Your program must print the following output:

  • During each iteration of the reroll phase:
    • The user's dice, formatted by the printDice function
    • The number of rolls remaining formatted as "X rolls remaining."
    • If the user enters an invalid command, the message "ERROR: INVALID COMMAND"
  • At the end of the reroll phase:
    • The user's dice, formatted by the printDice function
  • During the score phase:
    • A list of options produced by the printScores function
    • The message "ERROR: INVALID NUMBER" for each invalid input the user provides
    • The number of points received for the chosen category formatted as "You receive X points."

Additionally, your program should include reasonable prompts for information and any additional text that you think would make it easier for the user to interact with the program. It's up to you how much of this to include, what exactly to say, and how to format it. The tests will not check these prompts.

Functions

Your program must include all of the functions described below in addition to the functions required in Project 3.

rerollPhase

  • Return type: void
  • Parameters:
    • An int array representing a set of five dice

The user begins with 2 available rolls. For as long as the user has more than 0 rolls, this function must repeat the following process:

  • Print the user's dice and roll count
  • Prompt the user for a command
  • Execute the user's command:
    • r: the string of letters after "r " must be passed to the rollSomeDice method along with the dice array, and the roll count must be decreased by one
    • skip: the roll count must be set to zero
    • any other commands must result in an error message

Print the user's dice again after the above loop is finished.

scorePhase

  • Return type: void
  • Parameters:
    • An int array representing a set of five dice

Must perform the following process:

  • Print the score options and how many points each option is worth
  • Read integers from the user until they enter a number from one to six, displaying an error message for each invalid input
  • Calculate and print the score for the category chosen by the user

countDice

  • Return type: int
  • Parameters:
    • An int array representing a set of five dice
    • An int representing the die value to count

Returns the number of times the die value appears in the array.

printScores

  • Return type: void
  • Parameters:
    • An int array representing a set of five dice
