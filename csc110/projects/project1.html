Project 1: Character Sheet Part 1

This program will prompt the user for a few pieces of information about their character, calculate some additional stats, and then print a character sheet.

Primary Topics

  • Input
  • Output
  • Variables
  • Mathematical expressions

Prerequisites and Setup

  • Create a file named CharacterSheet1.java in your projects folder.

Tasks

Task 1: Guided Coding

Follow the steps below and write the program as directed. Test your code frequently, at the bare minimum performing every test given in the steps.

If your code is not working as expected, do not move on to the next step until you have fixed it. Moving on without finishing a step correctly will make the problem significantly harder to fix and likely add new problems as you continue working. If you disregard this advice, then you may get to the end of the steps and find it would actually be less work to start over from step 1 than to debug the code you just finished writing.

  1. I recommend using comments to divide your program into sections so that it's easier to stay organized. I'll be referring to three different parts of the program in these steps: input, calculations, and output. Before the input section, add comments with a title for the program and the author's name (that's you). Feel free to copy the template below and fill in the missing information.
// Project 1: Character Sheet Part 1

    //////////////////

    //////////////////

    // TODO: calculations

    ////////////
    // Output //
    ////////////

    // TODO: print character sheet
}
  1. In the input section, prompt the user for the following information in order: their name, their character's name, their character's level, and their character's ability scores (strength, dexterity, and constitution). Prompting for input takes 1-2 lines of code and involves several steps:
    • Declare a new variable to hold the input value. Give the variable a meaningful and descriptive name.
    • Call readln and include a prompt telling the user what information you expect them to type. You can word the prompt however you want; the test program will ignore your prompts when checking your program's output.
    • If you need your input as a number, remember to call Integer.parseInt or Double.parseDouble before readln, and place the readln call inside of the parenthesis after the parse function.
  2. Add some temporary print statements to the output section. Print the value of each variable you read during the input section. This will help with early testing. I recommend including the variable's name in its print statement: println("x = " + x); instead of println(x);.
  3. Test your program. You should see the first input prompt, and then the program should pause until you enter your input before moving to the next prompt. The program should repeat all of your input after you complete the final prompt.
  4. In the calculations section, calculate the character's HP (hit points), AC (armor class), and attack bonus. Each of these properties should be a new variable. Use the following formulas to calculate these properties:
    • The character's HP is equal to their level multiplied by their constitution score.
    • The character's AC is equal to ten plus their dexterity score.
    • The character's attack bonus is equal to their level plus their strength score.
  5. Add print statements for the new variables and test your program again. Pay attention to the values printed for the hp, ac, and attack bonus variables, and check the calculations yourself to confirm these are correct. If everything looks right, then this is a good time to delete or comment out the test print statements. We'll to write the actual output section next, so we shouldn't need the test output anymore.
  6. Print all of the information shown in the I/O Format section. Pay careful attention to spelling, capitalization, and the order in which this output appears. In order to pass the tests, it will have to match exactly with the expected output.
  7. Before you run my test program, try running your program yourself and make sure the output looks correct. It will be nice to see the whole program working, and it'll be faster to fix any obvious mistakes on your own than using the test program.

Additional Details

I/O Format

This program must meet the following I/O requirements:

  • The program accepts the required input in the order shown in the assignment and does not expect any additional input.
  • The program produces each line of required output in the order specified in the assignment.

The program may include additional output before, between, and after lines of required output. This additional output must be on a separate line from any required output.

Your program must accept the following user input:

  • The player's name (String)
  • The character's name (String)
  • The character's level (int)
  • The character's strength score (int)
  • The character's dexterity score (int)
  • The character's constitution score (int)

Your program must print the following output:

  • Player: <NAME>
  • Character: <NAME>
  • Level: <LEVEL>
  • HP: <HP>
  • AC: <AC>
  • Attack: +<ATTACK BONUS> (note the + before the attack bonus)
  • Str <STRENGTH SCORE>
  • Dex <DEXTERITY SCORE>
  • Con <CONSTITUTION SCORE>
