Project 1: Wheels Scene

Using the wheels library, create a scene involving several objects. The scene can be whatever you want as long as it's appropriate for a classroom setting.

The wheels library provides several classes to display simple shapes. You can use these to construct all of the objects in your scene. You can see a list of these shape classes in the wheels documentation on Blackboard.

Primary Topics

  • Wheels graphics library
  • Composition
  • Relative positioning

Setup and Prerequisites

Download the wheels library and documentation from Blackboard and learn how to import it into a project. Make sure you can run the following wheels program:

public class Main extends wheels.users.Frame {
    public static void main(String[] args) {
        new Main();
    }
}

This program should display a mostly empty window with a quit button.

Tasks

Task 1: Design

To start, design a scene with at least three different types of objects. For example, your scene might be a bird flying through the sky with the sun and some clouds in the background. The bird, sun, and clouds would be our three types of object. Once you have an idea, think about how you could create these objects using wheels shapes (rectangles, ellipses, lines, and rounded rectangles). Our sun, for example, might be an ellipse with several lines radiating from it. I recommend drawing a quick sketch of the objects using those shapes. This will help you figure out how to create the objects using code.

Once you've designed your scene, you need to design an object that functions as a component of another object in your scene. This component must appear multiple times in your code: either multiple times within the same object or in multiple types of object. Using our example, a single wing could be used as a component because our bird needs two of them. If we were going to include two different types of birds in our scene, then we might make a beak component and use it once in each bird class.

In a separate document, answer the following questions about your design. This is your design writeup, and it must be submitted with the project.

  1. Summarize the entire scene.
  2. Describe three types of objects in your scene. What does each object represent, and how will you construct it out of wheels shapes? Your scene can include more types beyond these three, but the three you describe must meet the following requirements:
    • The object is composed of at least three other objects (wheels shapes or component objects)
    • The object is represented by its own class in your program
    • The scene must include multiple instances of at least one object, so note an object that appears more than once in the scene
  3. Describe a component object that appears in the scene. What does this component object represent, how will you construct it out of wheels shapes, and what other object(s) will use it as a component? This component object must meet the following requirements:
    • The component is composed of at least three wheels shapes
    • The component is part of at least one type of object described in the previous question
    • The component is either used multiple times in one object (for example, a bird with two instances of a wing component) or in different objects (for example, an eagle and an owl that each have a beak component)

Finally, note that I expect you to design your own scene and write these descriptions yourself. Your scene should not be based on a classmate's scene or the example given in this section, and you are not allowed to use ChatGPT or another LLM to assist with this writeup.

Task 2: Program

Write a wheels program that displays your scene. It must include the following classes:

  1. A Main class that extends wheels.users.Frame and includes a main method that launches the program. Make sure you name this class correctly and leave it in the default package so that my script can run your program.
  2. One class for each type of object described in your design writeup. The classes should be named appropriately (upper camel case names that identify what type of object they represent).
  3. A class that defines the component described in your design writeup.

Feel free to include more than the five required classes if you want to add more to your scene.

Your three object types and your component must include the following constructors and methods:

  • A setLocation method that moves the entire object to the given location
  • A setColor method that changes at least one of the object's colors
  • A default constructor that initializes all of the object's components and calls the above setter methods
  • A parameterized constructor that accepts x, y coordinates for the object's location and at least one color; these parameters must be used to call the above setter methods

Task 3: Reflection

Answer the following questions after completing your program. This reflection can be in the same document as your design writeup as long as long as it is clearly separated from your answers to the design questions.

  1. Did you include any other setter methods in one of your classes to change its other properties (such as rotation or size)?
    • If yes, then describe that method and why you included it.
    • If no, then what additional property would you want to add a setter for? How would this help you improve your scene?
  2. Review your program and find a piece of code that was either challenging to write or that you think is an example of particularly good code (it's up to you what "good code" means, but you should be able to justify your opinion). Explain why you chose this code: why is it an example of good code or what was challenging about writing it? Include the code in your answer and note the line number(s) and file where it appears in your program.
  3. Review your program and find a piece of code that you think could be improved. What do you dislike or want to change about this code? If you were to spend more time on the project, how might you improve this code? To be clear, I'm not asking you how the program could be better. I don't want to know what feature you'd add or how you'd make it easier to use. I'm asking how part of the code could be better. This might mean shorter, easier to read, less repetitive, or something else that you value and think you could improve with additional time and/or programming knowledge. Include the code in your answer and note the line number(s) and file where it appears in your program.
