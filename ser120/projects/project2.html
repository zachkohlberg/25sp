Project 2: Interactive Scene

Using the wheels library, create a program with interactive elements. This can be a scene like in the previous project, or something completely different.

Add interaction to wheels objects by extending wheels shapes (such as Rectangle and Ellipse) and overriding one or more of the mouse methods (such as mousePressed, mouseReleased, mouseClicked, or mouseDragged).

Primary Topics

  • Composition
  • Inheritance
  • Interfaces
  • Polymorphism

Setup and Prerequisites

You should complete project 1 before starting this project. I also recommend using your scene from project 1 as a starting point for project 2.

Tasks

Task 1: Design

Design at least two interactive elements to include in your program. You can add some UI elements to the scene from your previous project, allow direct interaction with objects in the scene, or do something else entirely. To give a few examples:

  • A button that triggers some action when clicked
  • A slider that gradually adjusts some property of an object
  • A composite object that can be dragged around the screen
  • An object that responds differently depending on where it is clicked

In a separate document, answer the following questions about your design. This is your design writeup, and it must be submitted with the project.

  1. Describe your two interactive elements and how interacting with them affects the program. For an object to count, it must meet the following requirements:
    • The interaction involves mouse input
    • The object is visible
    • The interaction is not limited to a single wheels shape (for example, an ellipse that can be dragged around on its own is too simple to count)
    • The interaction causes a visible change in one or more objects
    • Each interaction must involve a different type of change (for example, moving, resizing, or changing an object's color)

Finally, note that I expect you to design your own objects and write these descriptions yourself. You objects should not be based on a classmate's objects, and you are not allowed to use ChatGPT or another LLM to assist with this writeup.

Task 2: Program

Write a wheels program that displays these objects. It must include the following classes:

  1. A Main class that extends wheels.users.Frame and includes a main method that launches the program. Make sure you name this class correctly and leave it in the default package so that my script can run your program.
  2. For each interactive object:
    • At least one subclass of a wheels shape that reacts to mouse input.
    • Another object that visibly responds to the mouse input.
    • An interface that links the wheels shape(s) to the object(s) that they control. The wheels shape(s) must have an association with this interface, not the other object. The other object must implement this interface.

You're free to include more classes in your project, but at a minimum you need a classes and interfaces that meet the requirements above.

Task 3: Reflection & UML

Answer the following questions after completing your program. This reflection can be in the same document as your design writeup as long as long as it is clearly separated from your answers to the design questions.

  1. For each of your two interactive objects, create a UML diagram that includes the following classes and relationships (don't include any class members in the diagram):
    • The interface used in the interaction
    • The wheels shape subclass(es) responsible for detecting mouse input
    • Any classes that implement your interface and the appropriate arrow for this relationship
    • Any associations or composition relationships between the above classes and interfaces
    • The superclass of each of the above classes and the appropriate arrow for this relationship
  2. Review your program and find a piece of code that was either challenging to write or that you think is an example of particularly good code (it's up to you what "good code" means, but you should be able to justify your opinion). Explain why you chose this code: why is it an example of good code or what was challenging about writing it? Include the code in your answer and note the line number(s) and file where it appears in your program.
  3. Review your program and find a piece of code that you think could be improved. What do you dislike or want to change about this code? If you were to spend more time on the project, how might you improve this code? To be clear, I'm not asking you how the program could be better. I don't want to know what feature you'd add or how you'd make it easier to use. I'm asking how part of the code could be better. This might mean shorter, easier to read, less repetitive, or something else that you value and think you could improve with additional time and/or programming knowledge. Include the code in your answer and note the line number(s) and file where it appears in your program.
